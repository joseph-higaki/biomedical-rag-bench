## Preliminary observations (hand-added)

> Added by hand below this run's auto-generated tables. Re-running `--run` regenerates
> everything above and drops this section. **n = 1 per type** — this is a smoke that the
> end-to-end loop works and a first directional read, **not** evidence. The real numbers
> come from the analysis notebook/dashboard over the full set × all retrievers.

**The H7 signal is visible.** Closed-book Haiku passes only the 0-hop memorized attribute
(HTR3B → chromosome 11) and fails every structural question (multi-hop, aggregative, set
intersection/difference, path existence). That is exactly H7's predicted shape — retrieval
unnecessary for a known 0-hop fact, necessary once the answer requires traversing the
graph. The crossover is the thing to measure properly; this just shows the baseline sitting
where the hypothesis expects.

**Two failure modes, both H7-relevant.** (1) *Hallucination* — types 02/03 returned
plausible-but-wrong lists (recall 1/11 and 5/18, with 29 and 14 spurious extras): the model
knows the neighborhood exists but not its exact membership. (2) *Refusal* — types 04–09
mostly answered "I don't have specific context…". Retrieval is the fix for both: supply the
membership, and the structural answers should appear.

### Caveats this run exposed (fix before reading closed-book as a knowledge measure)

1. **The prompt biases closed-book toward refusal — likely understates Haiku's knowledge.**
   The system prompt says "use only the Context; if no Context is given, answer from your own
   knowledge," yet the model repeatedly anchored on "I don't have specific context provided"
   and declined. So "1/9" probably under-reads what Haiku actually knows — the closed-book
   condition needs its own framing that doesn't prime context-dependence. **This is a
   validity issue for H7's closed-book arm, not a cosmetic one.** → prompt-tuning TODO.

2. **Verbosity hurt set-match precision.** Despite "be concise," answers led with markdown
   headers and preambles ("To answer this question accurately, I need to…"); those lines are
   parsed as claimed set members, inflating the "extra" count and depressing F1. Tighter
   output-format steering (or a list-only instruction for set questions) is a → TODO.

3. **Binary judge missed a refusal cue.** Type 08 ("I don't have specific context… about
   Testolactone") was scored "did not refuse." The negation regex has `doesn't` but not
   `don't` / `I don't` — a real gap in `eval/judge/deterministic.py`'s `_NEGATION`. → add
   those cues and re-examine (ties to the documented limit that true hallucination detection
   needs entity linking).
