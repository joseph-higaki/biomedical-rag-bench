## Decisions (and why)

- **Streaming transform, not in-memory.** 712 MB JSON + full `rdflib.Graph` won't fit in ~7 GB RAM. Stream in/out; memory bounded regardless of dataset size. Recorded as a provisional decision in the README. Revisit if too slow or escaping bugs accumulate.
- **pyoxigraph, not rdflib, for validation.** rdflib 7.6 has **no RDF-star support** (no `<<>>`, no `{| |}`, no `turtle-star` plugin). pyoxigraph parses Turtle-star, runs SPARQL-star, and stands in for GraphDB offline. rdflib dropped from deps.
- **URI namespaces.** Fixed `db:`/`do:`/`ncbigene:`/`uberon:` per CLAUDE.md; chose `go:`/`umls:`/`mesh:`/`ndfrt:` for the rest; **`pathway:` is project-minted** (`https://het.io/pathway/`) — no clean external vocab for `PC7_*` ids. Schema under `hetio:`. Edge properties via RDF-star annotations. Full table in `ingest/hetionet-data-notes.md` — worth a second look if a reviewer cares about vocab fidelity.
