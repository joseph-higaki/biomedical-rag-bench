## Decisions (and why)

- **Project CLAUDE.md for the uv runner preference, not a hook.** User asked
  what persistence artifact was right. Options were: project CLAUDE.md
  (instruction, version-controlled), a PreToolUse hook (harness-enforced,
  strongest), or global CLAUDE.md (cross-project). Chose project CLAUDE.md —
  sufficient for an instruction that travels with the repo and loads every
  session, without the complexity of a hook. Global CLAUDE.md rejected because
  uv is project-specific, not universal.
- **CPU-only torch for this dev box.** AMD Ryzen 7, no GPU, WSL2. The CUDA
  build adds ~1.5 GB of packages for zero gain here. `[tool.uv.sources]` with
  `explicit = true` pins torch to `https://download.pytorch.org/whl/cpu` without
  affecting other packages. The `all-MiniLM-L6-v2` model runs fine on CPU at
  smoke/sample scale.
- **Document the naive observation now, not at release.** The finding is useful
  before the full eval runs: it illustrates H2/H4 concretely and gives the user
  (no biotech background) a plain-English foothold for understanding what vector
  retrieval actually does. Put it at two levels — subsystem (`ingest/vector/README.md`)
  and project (`README.md`) — so it's visible whether someone is reading the
  architecture or the ingestion code.
