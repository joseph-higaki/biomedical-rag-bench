## Decisions (and why)
- **Embed hops in the retriever name, not a CLI factor.** The name is already the condition
  key (run_id, manifest, grouping); embedding keeps it one key with no manifest schema
  change and auto-namespaces result files. Naming `_<n>hop` (single trailing token), not
  `_n_hops`. Reserve name-encoding for curated comparison conditions, not arbitrary sweeps.
- **Targeted vector corpus (option A) over full or a parallelized fetcher.** Full is ~11 h
  (latency-bound ~0.7 entity/s); targeted (seeds + 1,500 distractors, ~1 h) makes vector
  fair for this eval now. Full evolving-baseline corpus = a later follow-up (parallelize).
- **Transient API error ≠ a verdict.** Isolate as an unscored row; don't let a network blip
  read as a wrong answer or destroy a 52-call run (the prior 2hop full run was lost that way).
