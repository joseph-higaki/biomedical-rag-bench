## Decisions (and why)
- **`context_tokens` is an offline proxy, not billed truth.** Exact token cost comes from the generator's `usage` metadata at step 5, in the generator's tokenizer. Proxy and generator tokenizer are different *currencies* → never subtract across them (`billed − proxy` is a units error). `traversal_info["context_tokenizer"]` tags the unit so the analysis layer can refuse mixed-currency math. The one legitimate decomposition is billed-only: `input(retriever) − input(closed_book)`, same model/question — which is exactly what the `closed_book` token baseline is for.
- **Generator ≠ judge ≠ "eval model".** Generator = model under test (answers); judge = scorer (deterministic for 9/10 types, LLM only for type 10). `GENERATOR_MODEL` is swappable across runs (small Llama → hosted), fixed within a run, logged. Multi-provider generator is a Strategy+Adapter behind a `Generator` Protocol + registry (step 5), normalizing each SDK's `usage` shape — no LiteLLM/orchestration dep (only ~3 providers, repo's hand-rolled ethos).
- **Graph retrieval: neighborhood now, text-to-SPARQL later.** `graph_neighborhood` (Mechanism 2, no LLM) isolates the *representation*; `graph_sparqlgen` (Mechanism 1, text-to-SPARQL) measures the realistic *system* but needs an LLM-in-retriever (step-5 infra) and confounds representation with query-gen skill. Two conditions, two research questions — both registered under one contract. Deferred sparqlgen needs only a schema *vocabulary* prompt (two `SELECT DISTINCT`), **not** OWL reasoning, so the Project 1 "reasoning stays empty" constraint holds. "Graph hallucinates less" is a hypothesis to measure (it relocates answer-hallucination into query-hallucination), not an assumption.
- **Factorial recording is co-produced.** Per-retrieval factors → `traversal_info` (retrievers); run-constant factors (generator/judge/embedding model, git SHA, seed, …) → a per-run manifest (harness, step 5). Authoritative home for the full factor list + manifest schema = `eval/README.md` (pending). Hypotheses (H7) are interaction effects, so every row must carry every factor's level.
- **One `retrievers/README.md`, sections per retriever — not a README per retriever.** Document at the granularity of the swap point (the contract), not the implementation; side-by-side comparison is the benchmark's point. A retriever earns its own folder/README only when it becomes a multi-file sub-package (the `eval/produce/` trigger).
- **Per-predicate fan caps, not a flat `LIMIT`.** On CDH1 the hundreds of `interacts`/`regulates` edges would crowd out the few answer-bearing `associates` edges under a global cap; per-predicate slots keep the answer reachable.
