## Decisions (and why)
- **Generator stays provider-agnostic (Strategy pattern).** User steer: keep message
  exchange, system prompt, token usage, *and tool usage* agnostic at the contract; all
  SDK/provider specifics live in the adapter. Added a neutral `tools=` in / `tool_calls`
  out surface (additive) for the deferred tool-using conditions (e.g. `graph_sparqlgen`).
- **Hosted Haiku for the first e2e** (over local Ollama / in-process transformers): fastest
  green loop, ~cents total, and it exercises the real billed-`usage` path the contract must
  normalize. The local model stays the documented iteration path (`.env.example` already
  names it).
- **git-SHA code-version manifest factor deferred** to a documented TODO. A bare short SHA
  that ignores a dirty working tree is false provenance — capture it properly (full SHA +
  dirty flag) or not at all. [user call]
- **Results → tracked markdown** (`eval/FINDINGS.md`) for offline review, separate from the
  gitignored machine-readable JSONL + manifest (the analysis-layer feed). Definitive metrics
  are the notebook/dashboard's job, not the harness's.
- **Ordering within step 5: judges → generator → loop.** Judges are deterministic, free, and
  locally smoke-testable (and match the README's stated step-5 smoke), so they came first;
  local-first discipline.
