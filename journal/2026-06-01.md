## Decisions (and why)
- **Declarative templates over code templates.** Inert, diffable, auditable —
  fits a structurally-defined benchmark; producer is the single interpreter.
- **Ground-truth query in a separate `.rq` file.** Buys back SPARQL tooling and
  lets the file run standalone in GraphDB Workbench / pyoxigraph.
- **`VALUES` block as substitution point** rather than blind string-replace: the
  file stays valid runnable SPARQL and the swap point is structurally defined.
- **Renamed "oracle" → "ground_truth"** repo-wide (README + CLAUDE.md): "oracle"
  collides with Oracle DB for a DB-literate audience. Clarified that ground truth
  has one source (graph traversal); future variation is query *language* per
  backend (e.g. Cypher for an LPG), not "source" — so future split is by dialect
  under `ground_truth/`, additive.
- **`.rq` extension** (W3C-standard) over `.sparql`.
